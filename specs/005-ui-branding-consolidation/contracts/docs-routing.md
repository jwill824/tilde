# Contract: Docs Site Routing

**Branch**: `005-ui-branding-consolidation` | **Date**: 2026-03-30

---

## Overview

This contract defines the expected URL resolution behavior for all internal navigation links on the tilde docs site hosted at `https://thingstead.io/tilde/docs/`.

---

## Base Path Contract

| Setting | Value |
|---------|-------|
| `site` (Astro config) | `https://thingstead.io` |
| `base` (Astro config) | `/tilde/docs` |
| Effective root URL | `https://thingstead.io/tilde/docs/` |

All internal links generated by Astro/Starlight MUST be prefixed with `/tilde/docs`.

---

## Fix Specification

1. Investigate whether `base: '/tilde/docs'` vs `base: '/tilde/docs/'` (trailing slash) resolves the Starlight sidebar link generation bug.
2. Check `@astrojs/starlight` version; apply patch or version bump if a known fix exists.
3. Verify all sidebar `slug` values in `astro.config.mjs` generate correct href values.
4. Run `astro build` and inspect `site/docs/dist/` for correct link hrefs before deploying.

---

## Acceptance Test

After the fix, navigate to `https://thingstead.io/tilde/docs/` and verify:

- [ ] Clicking "Installation" in sidebar → `https://thingstead.io/tilde/docs/installation/`
- [ ] Clicking "Getting Started" in sidebar → `https://thingstead.io/tilde/docs/getting-started/`
- [ ] Clicking "Configuration Reference" in sidebar → `https://thingstead.io/tilde/docs/config-reference/`
- [ ] Refreshing any sub-page retains the correct URL
- [ ] Logo/home link returns to `https://thingstead.io/tilde/docs/`
