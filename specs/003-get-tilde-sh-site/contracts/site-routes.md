# Contract: Site URL Routing

**Date**: 2026-03-29

---

## Domain Map

| Domain | Purpose | Deployment Source |
|--------|---------|-------------------|
| `thingstead.io/tilde` | Landing page + install script | `site/tilde/` (no build, copied to `dist/tilde/`) |
| `thingstead.io/tilde/docs` | Full documentation site | `site/docs/dist/` (Astro build, copied to `dist/tilde/docs/`) |

Both paths are served from a single Cloudflare Pages project named `thingstead`.
DNS: `thingstead.io` CNAME → `thingstead.pages.dev` (proxied, one record).

---

## thingstead.io/tilde Routes

| Path | Content-Type | Description |
|------|-------------|-------------|
| `/` | `text/html` | Landing page — install options, description, link to docs |
| `/install.sh` | `text/plain` or `application/x-sh` | Install shell script |

All other paths on `thingstead.io/tilde` → 404 (no docs content lives here).

### install.sh Content-Type

Cloudflare Pages serves files with extensions correctly. `install.sh` MUST be served
with a content type that does not cause browsers to execute it (i.e., `text/plain` is
acceptable; `text/html` is not). curl ignores content-type and pipes the response body.

---

## thingstead.io/tilde/docs Routes

| Path | Description |
|------|-------------|
| `/` | Docs home — overview and quick links |
| `/installation/` | All install methods (curl, npx, npm, future: brew) |
| `/getting-started/` | First run walkthrough |
| `/config-reference/` | `tilde.config.json` schema reference |
| `/404/` | Custom 404 page with search and nav |

All routes are generated by Astro + Starlight at build time as static HTML.
No server-side routing.

## HTTPS Contract

- Both domains MUST be served over HTTPS only
- HTTP requests MUST redirect to HTTPS (Cloudflare Pages handles this automatically)
- HSTS header SHOULD be enabled (Cloudflare Pages default)
- `install.sh` MUST only be fetchable over HTTPS — curl command uses `-fsSL` which
  follows redirects and fails on certificate errors

---

## Caching Contract

| Resource | Cache Policy |
|----------|-------------|
| `index.html` (landing) | Short cache (`Cache-Control: max-age=300`) — updates on deploy |
| `install.sh` | Short cache (`Cache-Control: max-age=300`) — must pick up new version tags |
| Docs HTML pages | Standard static (`Cache-Control: max-age=3600`) |
| Docs assets (images, CSS, JS) | Long cache with content hash in filename |

`install.sh` MUST NOT be cached aggressively — users re-running the script should
always get the version that resolves the latest npm release.
